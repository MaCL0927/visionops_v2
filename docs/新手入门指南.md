# VisionOps 新手入门指南

> 面向没有 ML/DevOps 基础的用户，手把手教你从零跑通整个平台。

---

## 这个平台是干什么的？

简单说：
1. 你在**电脑（服务器）上**训练一个能识别图片的AI模型
2. 训练好之后，把模型**推送到 RK3588 小板子**上运行
3. 板子就能实时识别摄像头拍到的内容
4. 平台自动帮你**追踪每次训练的效果**、**监控板子的运行状态**、**发现模型变差了自动重训练**

---

## 第一步：安装前置软件

你的电脑上需要安装这些东西（**只需要装一次**）：

### 1.1 安装 Docker Desktop（最重要）

Docker 是用来一键启动所有后台服务的工具。

👉 下载地址：https://www.docker.com/products/docker-desktop/

下载安装后，打开 Docker Desktop，等它显示 **"Engine running"**（绿色图标）再继续。

> 💡 如果你用的是 Windows，安装时会让你启用 WSL2，按提示操作就行。

---

### 1.2 安装 Python 3.11

👉 下载地址：https://www.python.org/downloads/

安装时**务必勾选** "Add Python to PATH"（添加到路径），否则后面命令会报错。

安装完后，打开命令行，验证：
```bash
python --version
# 应该显示：Python 3.11.x
```

---

### 1.3 安装 Git（可选，但推荐）

👉 下载地址：https://git-scm.com/downloads

---

## 第二步：启动平台服务

平台所有的后台服务（数据库、模型管理、监控）都用 Docker 一键启动。

### 2.1 打开终端，进入项目目录

**Windows 用户**：按 `Win + R`，输入 `cmd`，回车打开命令行，然后：
```cmd
cd C:\Users\yuxia\WorkBuddy\20260417111041\visionops
```

**Mac 用户**：打开终端，输入：
```bash
cd ~/WorkBuddy/20260417111041/visionops
```

---

### 2.2 启动所有服务

```bash
docker-compose up -d
```

第一次运行会下载镜像，需要等 **5-15 分钟**（取决于网速）。

看到类似下面的输出就说明成功了：
```
✔ Container visionops-postgres    Started
✔ Container visionops-minio       Started
✔ Container visionops-mlflow      Started
✔ Container visionops-prometheus  Started
✔ Container visionops-grafana     Started
✔ Container visionops-api         Started
```

---

### 2.3 验证服务正常运行

打开浏览器，访问这些地址：

| 服务 | 地址 | 用途 | 账号密码 |
|------|------|------|---------|
| **MLflow** | http://localhost:5000 | 查看每次训练的结果 | 无需登录 |
| **API 文档** | http://localhost:8000/docs | 后端接口说明 | 无需登录 |

## 第三步：安装 Python 依赖

回到命令行，在 `visionops` 目录下运行：

```bash
pip install -r requirements.txt
```

这会安装 PyTorch、MLflow 等所有需要的库，需要等几分钟。

> 💡 如果网络慢，可以加上国内镜像：
> ```bash
> pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
> ```

---

## 第四步：初始化项目结构

```bash
pip install dvc dvc-s3   # 先安装 DVC（数据版本管理工具）
python -m dvc init       # 初始化 DVC
```

然后配置 DVC 连接本地 MinIO：
```bash
dvc remote add -d minio s3://visionops-data
dvc remote modify minio endpointurl http://localhost:9000
dvc remote modify minio access_key_id minioadmin
dvc remote modify minio secret_access_key minioadmin123
```

创建必要的文件夹：
```bash
# Windows
mkdir data\raw data\processed models\checkpoints models\export models\metrics logs\retrain

# Mac/Linux
mkdir -p data/raw data/processed models/checkpoints models/export models/metrics logs/retrain
```

## 第五步：准备你的训练数据

把你的图片放到 `data/raw/` 文件夹里，按**类别分文件夹**：

```
data/raw/
├── 猫/              ← 放猫的图片
│   ├── cat001.jpg
│   ├── cat002.jpg
│   └── ...
├── 狗/              ← 放狗的图片
│   ├── dog001.jpg
│   └── ...
└── 鸟/              ← 放鸟的图片
    └── ...
```

> 💡 每个类别**至少准备 50 张图片**，越多效果越好。

---

## 第六步：配置训练参数

打开文件 `pipeline/configs/train.yaml`，根据你的需求修改：

```yaml
model:
  architecture: yolov8n   # 模型大小：yolov8n(最小最快) / yolov8s / yolov8m(最准但最慢)
  num_classes: 3           # ← 改成你的类别数量（比如猫狗鸟就填3）

train:
  epochs: 50               # 训练轮次，越多越准但越慢，新手先用50试试
  batch_size: 16           # 每批处理图片数，内存不够就改成8
  lr: 0.001                # 学习率，一般不用改
```

> 💡 `num_classes` 一定要和你的图片文件夹数量一致！

---

## 第七步：运行完整训练流水线

一条命令跑通所有步骤：

```bash
python -m dvc repro
```

这条命令会自动按顺序执行：

```
第1步：数据预处理  →  第2步：模型训练  →  第3步：评估效果  →  第4步：导出ONNX  →  第5步：注册模型
```

训练过程中你可以打开 **http://localhost:5000** 实时看训练进度和指标！

训练完成后，模型文件会保存在 `models/export/model.onnx`。

---

## 第八步：查看训练结果

打开 **http://localhost:5000**（MLflow），你会看到：

- 每次训练的**精度**（accuracy/mAP）
- 训练的**损失曲线**
- 模型的**参数配置**

右上角可以对比不同次训练的效果。

## 第九步：部署到 RK3588 板子

> ⚠️ **注意**：ONNX → RKNN 转换这一步需要在 **Linux 环境**下做，Windows 上需要先配置 Linux 虚拟机或者用 WSL2。
> 
> 如果你只是先在电脑上测试，可以跳过这步，直接用 ONNX 格式在电脑上推理。

### 9.1 配置设备信息

打开 `pipeline/configs/mlops.yaml`，填写你的 RK3588 设备信息：

```yaml
edge_devices:
  - id: rk3588-001
    host: 192.168.1.100    # ← 改成你的板子的IP地址
    port: 22
    user: root             # ← 登录用户名
    deploy_path: /opt/visionops/models/
```

### 9.2 在板子上初始化环境（只需做一次）

把项目里的脚本传到板子上，然后在板子上执行：
```bash
sudo bash setup_edge.sh http://192.168.1.xxx:8000 rk3588-001
```
（把 IP 换成你的服务器 IP）

### 9.3 推送模型到板子

```bash
python -m dvc repro   # 先确保模型已转换成 RKNN 格式
bash edge/deploy/push.sh models/export/model.rknn
```

### 9.4 在板子上测试推理

模型部署好之后，板子会自动启动一个推理服务。用以下命令测试：

```bash
# 发一张图片过去，看看识别结果
curl -X POST http://板子IP:8080/infer -F "file=@你的图片.jpg"
```

## 常见问题

### Q: `docker-compose up` 报错，说端口被占用
某个端口已经被其他程序用了。检查哪个端口冲突，关闭对应程序，或者修改 `docker-compose.yml` 里的端口号。

### Q: 训练时提示 GPU 不可用，怎么用 CPU 训练？
没有 GPU 也没关系，用 CPU 可以训练，只是慢一些。在 `train.yaml` 里加上：
```yaml
train:
  device: cpu
```

### Q: `pip install` 很慢或失败？
换国内镜像：
```bash
pip install -r requirements.txt -i https://mirrors.aliyun.com/pypi/simple/
```

### Q: RKNN 转换这步报错？
这步需要在 Linux 环境下运行，Windows 直接运行会跳过（生成占位文件），不影响其他步骤。

### Q: 如何重新训练（精度不满意）？
修改 `train.yaml` 参数后，运行：
```bash
python -m dvc repro --force
```
